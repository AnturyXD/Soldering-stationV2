# 焊台状态串口接口文档

**项目：** 智能焊台系统 V2  
**文档版本：** V1.0  
**日期：** 2026-06-08

## 1. 接口概述

本接口用于智能焊台温控板向 STM32 或其他上位机主动上报设备状态。

当前正式固件仅支持单向状态上报，不支持上位机向焊台下发控制指令。

适用固件：

```text
控温板/Firmware/1.7_uart_nonblocking/build/1.7_uart_nonblocking.ino.hex
```

## 2. 硬件接口

### 2.1 引脚定义

| 信号 | 焊台端 | 上位机端 | 说明 |
|---|---|---|---|
| UART TX | ATmega328P `PD1/TXD` | UART RX | 焊台状态数据输出 |
| GND | 焊台 GND | 上位机 GND | 通信双方必须共地 |

ATmega328P-AU/TQFP-32 的 `PD1/TXD` 为物理引脚 31。

### 2.2 电平转换

```text
ATmega328P PD1/TXD ── R19 10kΩ ──●── CN2-1 ── 上位机 RX
                                 │
                              R18 20kΩ
                                 │
                                GND

CN2-2 ────────────────────────── GND ── 上位机 GND
```

`●` 为分压点。理论输出高电平：

```text
Vout = 5V × 20kΩ / (10kΩ + 20kΩ) ≈ 3.33V
```

| 测量位置 | UART 空闲电压 | 发送时波形 |
|---|---:|---|
| PD1/TXD、R19 输入侧 | 约 5V | 0～5V |
| 分压点、CN2-1 | 约 3.3V | 0～3.3V |
| CN2-2 | 0V | 保持 0V |

## 3. 串口参数

| 参数 | 配置 |
|---|---|
| 波特率 | `9600 bps` |
| 数据位 | `8` |
| 停止位 | `1` |
| 校验位 | `None` |
| 硬件流控 | `None` |
| 数据编码 | ASCII |
| 报文结束符 | LF，即 `0x0A`，不发送 CR |
| 上报周期 | 约 1000ms |

## 4. 状态报文

### 4.1 报文格式

```text
$T,<current_temp>,<set_temp>,<power_percent>,<mode>\n
```

示例：

```text
$T,326,350,42,H\n
```

实际字节流的结尾为一个 LF 字节，不包含字符 `\` 和 `n`。

### 4.2 字段定义

| 序号 | 字段 | 类型 | 说明 |
|---:|---|---|---|
| 1 | `$T` | 固定字符串 | 状态报文帧头 |
| 2 | `current_temp` | 十进制整数 | 当前显示温度，单位 °C；温度检测异常时发送 `999` |
| 3 | `set_temp` | 十进制整数 | 当前设定温度，单位 °C |
| 4 | `power_percent` | 十进制整数 | 加热功率百分比，范围 `0～100` |
| 5 | `mode` | 单个 ASCII 字符 | 当前设备运行状态 |
| 6 | `\n` | `0x0A` | 报文结束符 |

### 4.3 运行状态码

| 状态码 | 名称 | 说明 |
|---|---|---|
| `E` | Tip Error | 未插入烙铁头或温度检测异常 |
| `O` | Off | 停机模式 |
| `S` | Sleep | 睡眠模式 |
| `B` | Boost | Boost 加热模式 |
| `W` | Working | 温度接近设定值并处于稳定工作状态 |
| `H` | Heating | 正在加热 |
| `L` | Holding | 低功率维持或保温状态 |

## 5. 上位机解析要求

1. 使用循环缓冲区持续接收串口数据。
2. 使用 LF（`0x0A`）作为完整报文结束符。
3. 丢弃不以 `$T,` 开头的行。
4. 按英文逗号分割后必须得到 5 个字段。
5. 校验三个数值字段能否正常转换为整数。
6. 校验 `power_percent` 是否位于 `0～100`。
7. 校验 `mode` 是否属于 `E/O/S/B/W/H/L`。
8. 接收缓存建议不少于 64 字节。

不要因收到任意噪声字节就判断设备在线。只有收到格式合法的完整状态帧后，才能刷新在线状态。

## 6. 在线状态判断

| 状态 | 推荐判定规则 |
|---|---|
| 在线 | 收到任意一帧格式合法的 `$T` 状态报文 |
| 离线 | 连续 3 秒未收到合法 `$T` 状态报文 |
| 通信异常 | 持续收到数据，但连续出现无法解析的报文 |

推荐流程：

```text
收到串口字节
    ↓
按 LF 分帧
    ↓
检查帧头、字段数量和字段范围
    ↓
合法帧：更新设备状态并刷新在线超时计时器
非法帧：丢弃，不刷新在线状态
```

## 7. STM32 C 语言解析示例

以下函数用于解析已经接收到的一行完整报文。串口接收建议使用中断或 DMA，不要在中断中执行 `sscanf()`。

```c
#include <stdbool.h>
#include <stdio.h>
#include <string.h>

typedef struct {
    int current_temp;
    int set_temp;
    int power_percent;
    char mode;
} SolderStationStatus;

bool ParseSolderStationFrame(char *line, SolderStationStatus *status)
{
    if (strncmp(line, "$T,", 3) != 0) {
        return false;
    }

    if (sscanf(line, "$T,%d,%d,%d,%c",
               &status->current_temp,
               &status->set_temp,
               &status->power_percent,
               &status->mode) != 4) {
        return false;
    }

    if (status->power_percent < 0 || status->power_percent > 100) {
        return false;
    }

    return strchr("EOSBWHL", status->mode) != NULL;
}
```

## 8. 报文示例

| 报文 | 含义 |
|---|---|
| `$T,326,350,42,H\n` | 当前温度 326°C，设定 350°C，功率 42%，正在加热 |
| `$T,350,350,5,W\n` | 当前温度达到设定值，处于稳定工作状态 |
| `$T,999,350,0,E\n` | 未插入烙铁头或温度检测异常 |
| `$T,150,150,0,S\n` | 设备处于睡眠模式 |

## 9. 测试与验收

| 测试项 | 验收标准 |
|---|---|
| UART 基本通信 | 使用 `9600 8N1`，每约 1 秒收到一帧合法状态报文 |
| 电平检查 | PD1 空闲约 5V；CN2-1 空闲约 3.3V |
| 异常状态 | 不插入烙铁头时收到 `mode=E`、`current_temp=999` |
| 在线判断 | 收到合法帧后显示在线；超过 3 秒未收到合法帧后显示离线 |
| 异常数据处理 | 噪声或格式错误报文不会更新设备状态 |

## 10. 当前限制与扩展建议

- 当前协议仅支持焊台主动上报，不支持上位机控制焊台。
- 当前报文不包含校验和；短距离板间通信可通过格式校验和超时机制处理异常。
- 后续扩展协议时，建议新增独立帧头或协议版本字段，不要改变现有 `$T` 报文字段顺序。
- 如果后续增加双向通信，必须将上位机 TX 电平限制在 ATmega328P 可接受范围内，并定义命令确认与超时机制。
