# 焊台拓展板上位机方案说明 V0.3

## 目标

V0.3 上位机仍采用离线 HTML 文件，不依赖云服务或桌面安装程序。浏览器打开本地页面后，通过 ESP-01S 提供的 WebSocket 连接 STM32 拓展板。

```text
浏览器离线 HTML
  |
  | WebSocket ws://设备IP:81/
  v
ESP-01S WebSocket Bridge
  |
  | UART 115200
  v
STM32F103C8T6 拓展板
```

## 文件

| 文件 | 作用 |
|---|---|
| `Software/SmartSolder_Host.html` | 离线网页上位机 |
| `Software/ESP01S_WebSocket_Bridge/ESP01S_WebSocket_Bridge.ino` | ESP-01S 桥接固件 |
| `Software/ESP01S_WebSocket_Bridge/README.md` | ESP 烧录和协议说明 |

## 网页显示内容

- 设备连接状态
- 当前手动/自动模式
- 下位机在线状态
- 当前烙铁温度
- 三路负载状态
- 温度曲线
- ESP / WebSocket / 调试信息
- 运行日志

## 网页控制对象

| target | 功能 | STM32 继电器 |
|---|---|---|
| `light` | 照明灯 | `PB13` 第一路 |
| `fan` | 排风扇 | `PB12` 第二路 |
| `aux` | 第三路预留 | `PB14` 第三路 |
| `mode` | 模式切换，`0=手动`，`1=自动` | 非继电器 |
| `all` | `value=0` 关闭全部负载 | 三路全部关闭 |

## WebSocket 到 STM32 的命令

网页发送 JSON 到 ESP：

```json
{"type":"cmd","seq":"1","target":"light","value":1}
```

ESP 转换成 UART 文本行给 STM32：

```text
CMD seq=1 target=light value=1
CMD seq=2 target=fan value=0
CMD seq=3 target=aux value=1
CMD seq=4 target=mode value=1
CMD seq=5 target=all value=0
```

## STM32 到网页的状态

STM32 发送：

```text
STAT {"fw":"V0.3","mode":"MANUAL","espOnline":1,"webOnline":1,"lowerOnline":1,"temp":320,"setTemp":350,"vin":24,"presence":{"enabled":0,"detected":0},"relay":{"light":1,"fan":0,"aux":0},"runTimeMs":12345,"ip":"192.168.1.88","debug":"T:320,S:350,V:24"}
```

ESP 原样包装后推送给网页：

```json
{"type":"state","data":{"fw":"V0.3","mode":"MANUAL","relay":{"light":1,"fan":0,"aux":0}}}
```

## 模式说明

| 模式 | 行为 |
|---|---|
| 手动模式 | 网页可以控制三路负载 |
| 自动模式 | 保留人体传感器联动入口，V0.3 默认不启用 |

V0.3 中人体传感器未接入，自动联动默认关闭。进入自动模式后，普通负载命令会返回：

```text
ACK seq=x ok=0 reason=AUTO_RESERVED
```

`all=0` 仍可用于关闭全部负载。

## 调试建议

1. 先确认 ESP HTTP 页面可访问：`http://设备IP/`
2. 再用 HTML 连接：`ws://设备IP:81/`
3. 查看网页日志区是否出现 `hello`、`state`、`ack`
4. 查看 STM32 调试串口是否周期输出 `[APP]`
