# 拓展板 V0.3 项目进度与功能说明

日期：2026-06-04

## 本次完成内容

### 1. 固件工程

```text
Firmware/SmartSolder_Extension/C8T6/USER/SmartSolder_Extension.uvprojx
```

工程只保留 V0.3 需要的模块：

### 2. 三路继电器规范

V0.3 已按新要求固定继电器顺序：

| 继电器 | 引脚 | 功能 |
|---|---:|---|
| 第一路 | `PB13` | 照明灯 |
| 第二路 | `PB12` | 排风扇 |
| 第三路 | `PB14` | 预留 |

继电器默认为高电平触发，上电初始化后全部关闭。

### 3. 下位机串口

已实现 `USART3`：

```text
PB10 / USART3_TX
PB11 / USART3_RX
115200 bps
```

当前兼容报文：

```text
T:320,S:350,V:24
```

只要收到下位机短报文，就认为下位机在线；超过超时时间未收到则判定离线。

### 4. 调试串口

已实现 `USART1`：

```text
PA9 / USART1_TX
PA10 / USART1_RX
115200 bps
```

调试输出采用 DMA 发送队列。队列满时丢弃新字符，不阻塞主循环。

### 5. 人体传感器接口

`PA1` 已作为人体传感器输入初始化。

V0.3 默认：

```c
#define PRESENCE_FEATURE_ENABLE 0
```

因此自动模式策略入口已保留，但不会实际根据 PA1 控制继电器。后续接入传感器并确认高低电平后，只需要改配置即可启用。

### 6. 手动/自动模式

当前支持两种模式：

| 模式 | 说明 |
|---|---|
| 手动模式 | 网页控制三路负载 |
| 自动模式 | 保留人体传感器联动入口，V0.3 默认不启用 |

自动模式未启用传感器时，负载命令会返回：

```text
ACK seq=x ok=0 reason=AUTO_RESERVED
```

`all=0` 仍可关闭全部输出。

### 7. TFT 屏幕

已改为：

```text
SPI1 硬件 SPI
DMA1_Channel3 发送
PA8 / TIM1_CH1 背光 PWM
```

屏幕显示内容：

- 固件版本
- 当前模式
- IP / Web 连接状态
- 三路负载状态
- 下位机温度
- 温度曲线
- 在线动画

屏幕刷新策略：

- 初始化时画静态框架
- 状态变化时局部刷新
- 不在主循环中反复整屏清空
- 大块区域支持 DMA 分行刷新

### 8. 离线 HTML 上位机

已重写：

```text
Software/SmartSolder_Host.html
```

功能：

README 已更新到 V0.3 协议。

## 资源与风险

- STM32F103C8T6 SRAM 只有 20KB，V0.3 未使用整屏 framebuffer。
- ST7789 DMA 使用一行缓冲：`240 * 2 = 480 bytes`。
- 调试串口 TX 队列使用 1024 bytes。
- ESP 串口 TX 队列使用 768 bytes，避免状态上报阻塞主循环。
- ESP/下位机串口接收使用小型行缓冲队列。
- 屏幕图形目前使用轻量 5x7 英文字体，中文显示尚未加入字库。

## 需要后续确认

4. 下位机是否固定使用 `T:xxx,S:xxx,V:xxx` 格式。
5. 是否需要在 STM32 侧下发控制命令到 ATmega。
6. 是否需要加入中文字库，或者只在网页端显示中文。
